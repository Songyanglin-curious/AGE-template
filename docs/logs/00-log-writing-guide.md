# 开发日志编写指南

开发日志条目按日期组织，每日一个文件。

## 目的

每条每日日志记录简短的标注说明，内容包括：

- 添加或更新了何种文档
- 做出了什么设计决策
- 下一步计划是什么
- 日后需记住，但不适合放入正式架构文档的小上下文

## 规则

- **每日一个文件** - 同一天的所有工作归入同一文件
- **追加新条目** - 在文件顶部添加新的 `### YYYY-MM-DD` 段（倒序时间）
- **保持条目简短** - 优先使用要点列表，链接到主文档或代码路径
- **非事实来源** - 此为轻量上下文，并非规范性架构
- **链接到正式文档** - 引用设计决策时，链接到架构文档或代码路径
- **将日志文件视为仅追加历史** - 文件长度本身不算缺陷；勿因每日日志超出有效文档的规模指导而标记问题

## 路径约定

- `docs/logs/{year}/{month}-{day}.md`

## 条目格式

```markdown
# Development Log - YYYY-MM-DD

### YYYY-MM-DD

- Brief description of what happened.
- Link to doc or code path: `docs/architecture/xxx.md` or `src/foo/bar.ts:42`
- Key decision: ...
- Next step: ...
```

## 添加新条目

1. 打开 `docs/logs/{year}/{month}-{day}.md`（若不存在则创建）
2. 在文件顶部，于任何现有条目之前，添加一个 `### YYYY-MM-DD` 段
3. 编写新的要点
4. 若当天已有更早的条目，则在空行分隔符之后追加
