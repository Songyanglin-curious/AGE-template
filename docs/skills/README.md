# 技能索引

使用此目录存储可复用的提示词和工作流手册。

这些不是一次性聊天消息，而是可复用的仓库记忆。

技能应主要捕捉可复用的工作方法、评审方法或审计方法。不要将技能用作需求真相、设计真相或架构真相的替代品。

技能库不是吸引子。如果不通过 `AGENTS.md`、`docs/index.md`、活动的需求和所有者文档进行路由，庞大的技能库通常会退化为结构化的氛围编码。

这些提示词是复制项目的通用默认值。在复制模板后，必须根据项目的真实所有者文档、保护区域、验证栈、命名约定、已知故障模式和误报容忍度对其进行定制。

## 技能路由规则

在选择技能之前：

1. 首先阅读相关的需求和所有者文档。
2. 使用 `AGENTS.md` 对任务类型进行分类。
3. 通过匹配工作方法而非仅根据业务标签来选择技能。
4. 如果有多个技能可能适用，请请独立的子代理或评审者在实施前做出选择。
5. 如果没有现成技能明确适用，记录 `Skill: none` 并继续使用正常的文档驱动工作流。
6. 对于非平凡的计划，在计划中记录技能选择的依据和评审结果。

不要添加宽泛的业务场景技能来替代项目特定的所有者文档。如果某个场景频繁出现，首先检查路由、所有者文档或计划指南是否缺失。仅在可复用的工作方法稳定后才提升为技能。

## 技能注册表

| 技能                                      | 适用情况                                     | 误用情况                     | 所需输入                                              | 期望输出                           |
| ----------------------------------------- | -------------------------------------------- | ---------------------------- | ----------------------------------------------------- | ---------------------------------- |
| `document-audit-prompt.md`                | 需求、设计或架构文档可能不完整或不一致       | 任务是琐碎且局部的           | 目标文档路径，相关输入或所有者文档                    | 审计发现和修订目标                 |
| `plan-audit-prompt.md`                    | 非平凡计划在实施前已准备好接受挑战           | 尚无计划                     | 计划文件，相关需求和所有者文档                        | 通过/不通过审计及具体问题          |
| `closure-audit-prompt.md`                 | 实施声称完成并需要独立的收尾评审             | 工作仍在进行中               | 计划、验证证据、相关变更文档                          | 收尾裁决和剩余差距                 |
| `requirement-gap-retrospective-prompt.md` | 已完成的工作仍未达到期望且需求流水线需要诊断 | 需求仍在起草中               | 原始输入、需求/讨论文档、交付结果                     | 回顾发现和流程纠正                 |
| `multi-dimensional-audit-prompt.md`       | 高风险工作需要同时从多个维度进行质疑         | 单一目标的审计已足够         | 相关需求/所有者文档、计划或变更区域、验证证据         | 按维度分组发现                     |
| `open-ended-audit-prompt.md`              | 隐蔽问题可能存在于常规检查清单之外           | 工作仅需要窄范围的结构化审计 | 相关需求/所有者文档、计划（如有）、日志、活动变更代码 | 对抗性发现和未知风险备注           |
| `index-routing-audit-prompt.md`           | 文档索引或目录结构需要路由有效性审查         | 索引没有路由角色或是琐碎的   | 顶级索引、子索引、目标文件                            | 覆盖率表、角色测试结果、结构性发现 |
| `bug-diagnosis-prompt.md`                 | 缺陷确实存在但根本原因尚未证实               | 缺陷已经明显且是局部的       | 缺陷报告、所有者文档、复现路径、验证命令              | 确认的原因和证明路径               |
| `code-quality-audit-prompt.md`            | 审查代码的行为风险和实现质量                 | 只需要格式或琐碎的小问题     | 变更文件、所有者文档、测试或验证证据                  | 按严重性排序的发现                 |
| `code-refactor-discovery-prompt.md`       | 在重构前需要发现结构性清理候选项             | 结构性目标已达成一致         | 目标范围、所有者文档、当前代码                        | 排序的重构候选项                   |
| `code-refactor-prompt.md`                 | 任务是保行为的结构性重构                     | 任务改变了支持的行为         | 目标范围、不变量、验证命令                            | 安全的重构执行和证明               |
