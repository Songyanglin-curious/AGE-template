# 代码重构发现提示

在变更结构之前，使用此技巧发现值得重构的目标。

## 扫描维度

1. 过大的文件
2. 重复或未收敛的逻辑
3. 隐藏的跨模块耦合
4. 单个模块中混杂的职责
5. 脆弱的异步或生命周期模式
6. 层间的边界违例
7. 镜像实现而非契约的测试
8. 陈旧的兼容性残留
9. 变更区域周边的文档/代码偏离

## 评审规则

- 不要仅因为代码显得臃肿就提出重构
- 优先考虑会反复出现的结构性风险
- 将保持行为的重构与契约变更区分开
- 明确具体的收益：拆分、合并、隔离、统一或删除

## 期望输出

- 带有证据的候选列表
- 每个候选对象的重要性说明
- 建议的执行顺序
- 当没有任何值得重构的目标时，明确说明理由
