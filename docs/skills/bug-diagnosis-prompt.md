# 缺陷诊断提示

当任务为缺陷调查，且失败原因尚未由明显的局部缺陷解释时，使用此技能。

此技能适用于在修复代码前，通过可重现的反馈循环找到真正原因。

## 适用场景

- 缺陷报告真实存在，但出现故障的层次不明确
- 症状可能源自另一子系统的下游效应
- 问题具有间歇性、状态相关或时间敏感性
- 若不遵循此流程，最可能的初步修复方案将沦为猜测

## 不适用场景

- 缺陷已还原为明确的一行代码缺陷
- 任务主要是功能开发，而非诊断

## 所需输入

- 缺陷报告或观察到的症状
- 相关责任人文档
- 复现环境或最接近的可用工具套件
- 当前验证命令

## 执行模式

1. 建立紧密的反馈循环。
2. 复现故障，并缩减为最小的可靠案例。
3. 列出 3-5 条可证伪的假设。
4. 运行针对性实验，快速排除假设。
5. 若可行，在修复前添加或更新回归验证证明。
6. 应用最小且正确的修复。
7. 重新运行验证，并移除临时性检测工具。

## 输出

- 已确认的根本原因
- 被否决的假设及其否决原因
- 回归验证证明路径
- 残余风险或待办事项
