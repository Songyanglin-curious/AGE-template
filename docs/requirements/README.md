# 需求索引

使用本目录存放可供实施的需求综合文档。

请勿直接将原始的 PM 聊天记录、复制的文章或原型笔记堆放于此。此类内容应首先归入 `docs/input/`。

## 规则

仅当内容足够清晰，能够指导设计或实施时，才移入此处。

这有助于保持原始输入与已确定的需求意图之间的界限。

## 建议文件

- `00-requirement-synthesis-guide.md`
- `product-scope.md`
- `mvp.md`
- `feature-001-<name>.md`
- `2026-05-21-user-management.md`
- `2026-05-21-order-refund-flow.md`

建议：

- 对于长期使用的基线文件（如 `product-scope.md` 和 `mvp.md`），使用稳定的文件名
- 对于与特定工作轮次相关的临时需求综合文件，使用带日期的文件名
