## 更新规则

当更改创建了新的主要入口点、移动了公共代码、添加了新的测试位置或反复导致代理重新发现同一路径时，更新此文件。

如果列出的路径缺失、占位符仍存在，或实际导入与此地图矛盾，请勿将地图视为权威。请以实时仓库为准进行验证，然后在实施前更新地图或将相应行标记为低置信度。

如果 `Last Verified` 对于项目节奏而言已过时、发生于重大结构变更之前，或者任务触及所列路径的边界，请在依赖该行之前验证实时仓库。低置信度行不会阻止实时验证后的低风险工作，但涉及受保护区域、迁移或跨模块的工作应在实施前更新该行。
