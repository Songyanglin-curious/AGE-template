# 上下文索引

## 目的

`docs/context/` 存放着 AI 代理在进行非微小更改之前应阅读的简短强制性上下文。

此目录有意保持精简。它之所以存在，是因为隐藏在 `docs/references/` 下重要项目规则容易被 AI 代理忽略。

## 请先阅读这些内容

1. `project-context.md`
2. `ai-autonomy-policy.md`
3. `codebase-map.md`
4. `source-of-truth-and-precedence.md`
5. `conventions.md`

## 此处应包含的内容

- 每个 AI 会话都应了解当前项目上下文
- AI 何时可以继续、制定计划、提问、研究或停止的自主权规则
- 紧凑的代码库路由，防止重复探索
- 单一事实来源与所有者文档优先级规则
- 项目范围的约定，而非仅可选的查阅材料

## 此处不应包含的内容

- 冗长的设计文档
- 一次性分析
- 日常日志
- 执行计划
- 详细的参考表格

请将它们放在各自的所属目录中，仅在需要时从上下文中链接到它们。
