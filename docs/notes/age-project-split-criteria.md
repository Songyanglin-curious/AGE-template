# AGE 项目拆分判据

## 核心原则

**实质大于形式。**

判断拆不拆项目的标准不是文件夹个数，不是部署单元数量，而是：

> **这些部分有独立的吸引子（独立的产品生命周期、独立的 backlog、独立的验证节奏）吗？**

有 → 拆。没有 → 不拆。

---

## 三问法

问三个问题来定性：

### 问题 1：有独立 backlog 吗？

- 各部分的 backlog 由不同 PM / 不同团队梳理 → 拆
- 同一个需求文档管所有部分 → 不拆

### 问题 2：验证和部署独立吗？

- 某部分可以上线而其他部分不跟着 → 拆
- 每次上线所有部分一起发 → 不拆

### 问题 3：接口是内部实现细节还是已发布的外部契约？

- 调用方唯一且和被调用方是同一团队 → 内部细节，不拆
- 调用方多个或不同团队维护 → 已发布契约，拆

---

## 决策对照表

| 信号 | 趋向：单项目 | 趋向：多项目 |
|------|:-----------:|:----------:|
| 同一个 PM / 同一个需求文档 | ✓ | |
| 各部分 PM 不同、backlog 无关 | | ✓ |
| 一次上线所有部分一起发 | ✓ | |
| 各部分独立发布、独立版本号 | | ✓ |
| API 调用方唯一、同团队 | ✓ | |
| API 多调用方、不同团队 | | ✓ |
| 共享同一个数据库 | ✓ | |
| 各管各的数据库，通过 API 交互 | | ✓ |
| 共享一套 CI/CD 流水线 | ✓ | |
| 独立 CI/CD、独立部署 | | ✓ |
| 改一部分极大概率要改其他部分（强内聚） | ✓ | |
| 一部分可长时间不动而其他快速迭代 | | ✓ |

## 典型案例

### 案例 1：前后端分离管理网站

```
my-app/
  frontend/
  backend/
```

**判据**：同一产品、同一 backlog、一起上线、一起验证。

**结论**：一个 AGE 项目。前后端是 `architecture/module-boundaries.md` 中的内部边界。

`project-context.md` 示例：

```markdown
## 当前技术基线
- 前端：React 18 + TypeScript（`frontend/`）
- 后端：Go + Gin（`backend/`）
- 共享契约：`api-schema/`
```

### 案例 2：主站 / 网站 / 站端 强内聚三层

**判据**：同一套需求驱动、一起上线、API 调用方唯一且同团队。

**结论**：一个 AGE 项目。三层是 `module-boundaries.md` 中的内部边界，记录各层职责和共享契约。

### 案例 3：多团队独立维护的微服务

**判据**：各自 PM、独立 backlog、独立部署、API 被多方调用。

**结论**：多项目，用两层结构管理。

## 多项目时的两层结构

```
方案仓库 (AGE 模板)     ← 拥有跨模块吸引子
  ├── 项目仓库 A       ← 拥有模块 A 的吸引子和实现
  ├── 项目仓库 B       ← 拥有模块 B 的吸引子和实现
  └── 项目仓库 C
```

### 系统仓库拥有：

- 项目愿景（`project-vision.md`）
- 模块边界定义（`module-boundaries.md`）：各模块职责、接口契约
- 跨项目约束（共享认证模型、数据格式）
- 全局输入（最顶层 PM 需求）
- 全局角色/权限定义

### 各项目仓库拥有：

- 本模块实现代码
- 本模块设计基线（`design/app-overview.md`）
- 本模块技术基线（继承系统约束，细化本模块决策）
- 本模块 backlog、plans、audits、logs

### 三层交互规则

1. **系统仓库的边界定义是项目仓库的硬约束**。项目仓库不重复全局规则，引用并细化。
2. **跨模块需求先进系统仓库**。在系统仓库的 `input/` → `discussions/` → `requirements/` 处理完，更新 `module-boundaries.md`，再通知各项目。
3. **冲突时系统仓库优先**。若项目仓库想偏离接口契约，必须先改系统仓库。

### 系统仓库的 `project-context.md` 特点

```markdown
## 当前技术基线
- 前端：无（本仓库不含实现代码）
- 后端：无（本仓库不含实现代码）
- 数据库：参见各项目仓库

## 验证命令
- 契约一致性检查：<Schema 校验命令>
- 跨模块影响分析：人工审查
```
