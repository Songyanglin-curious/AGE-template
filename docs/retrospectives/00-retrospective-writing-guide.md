# 回顾式写作指南

## 目的

当团队需要理解为何某项交付在实施工作完成后仍未达到预期时，使用 `docs/retrospectives/`。

## 适用场景

- 原型与实现之间的偏差超出预期
- 需求来源不够充分
- PM 到开发者的交接跳过了分析工作
- 相同的过程缺陷反复出现
- 应当提炼出新的可复用技能或审计提示

## 内容要点

- 原始来源输入
- 缺失或误导的部分
- 仅在实施过程中才被发现的问题
- 下次流程中哪些环节应该前置
- 应当创建哪些新的技能、审计提示或工作流规则

## 文件命名指导

建议使用带日期的文件名：

- `docs/retrospectives/YYYY-MM-DD-topic.md`
